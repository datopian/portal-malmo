**Accessibility for opendata.malmo.se**

The City of Malmö is behind this website. We want as many people as possible to be able to use it. This document describes how opendata.malmo.se complies with the Swedish Act on Accessibility of Digital Public Services, any known accessibility issues and how you can report deficiencies to us so that we can fix them.

**How accessible is the website?**

We are aware that parts of the website are not fully accessible. See the section on inaccessible content below for more information.

**What can you do if you cannot use parts of the website?**

If you need content from opendata.malmo.se that is not available to you, but is exempt from the scope of the law as described below, you can send an email to [opendata@malmo.se](mailto:opendata@malmo.se).

**Report deficiencies in website accessibility**

We strive to improve the accessibility of the website. If you discover any problems that are not described on this page, or if you believe that we are not meeting the legal requirements, please let us know by email at opendata@malmo.se so that we know that the problem exists.

**Supervision**

The Digital Management Authority, Digg, is responsible for supervising the Act on Accessibility to Digital Public Services. You can report to Digg if you think that our digital service has accessibility deficiencies.

You can also report to Digg if you think that our assessment of what is unreasonably burdensome should be reviewed, if you think that our accessibility statement has deficiencies or if you think that we have not handled your request for accessibility correctly.

**Technical information about website accessibility**

This website is partially compliant with the Act on Accessibility to Digital Public Services, due to the deficiencies described below.

**Content that is not accessible**

The content described below is not fully accessible in one way or another.

**Lack of compliance with legal requirements**

* Visually designed headings are not coded with heading elements  
* Content in English is not marked up correctly in the code  
* Heading structures lack heading levels  
* Visual lists lack correct semantic coding  
* Information about search results is not marked up as a status message  
* Decorative vector graphics are not hidden from assistive devices  
* Some text against a background has too low contrast  
* Messages about missing search results do not help users correct their search  
* Filtering changes the content in an unexpected way and hides other filters  
* The website does not respect the user's dark mode setting

**Content not covered by the law**

The content described here is not fully accessible, but is exempted according to Section 9 of the Act on Accessibility to Digital Public Services.

* Images of maps are not described with alt texts
